Roles Endpoint

CRUD de perfis de acesso via Identity · Rota base: /api/roles

Visão geral
/api/roles
IdentityRole
Objetivo: expõe CRUD de perfis (roles) via RoleManager<IdentityRole>. Usa o valor padrão de _settings.Theme.Role para bloquear exclusão do papel padrão.
Endpoints
GET
Listar — retorna todas as roles (recordsTotal/recordsFiltered).
GET
Obter por id — /api/roles/{id}, retorna IdentityRole.
POST
Criar — body form-data com campos de IdentityRole. Server-side gera Id e ConcurrencyStamp.
PUT
Atualizar — body form-data de IdentityRole; usa UpdateAsync.
DELETE
Remover — body form-data de IdentityRole; bloqueia exclusão se model.Name == _settings.Theme.Role. Retorna 204 ou erro do Identity.
Fluxo e validações
  • Listagem: usa AsNoTracking(); responde em envelope { data, recordsTotal, recordsFiltered } (padrão DataTables).
  • POST: IDs e concurrency são sempre novos GUIDs; responde 201 com Location para Get.
  • DELETE: regra de negócio simples para impedir remoção do papel padrão configurado em SmartSettings.
Segurança: Endpoint está como [ApiController]/ControllerBase. Confirme políticas de autenticação/autorização na pipeline global (não definidas aqui).
Exemplo de requisição (cURL)
curl -X POST https://seu-host/api/roles ^ -F "Name=Operador" ^ -H "Authorization: Bearer <token>"

Ajuste o header de autorização conforme o esquema ativo (Bearer/Cookie/etc.).
