Users Endpoint

Gestão de usuários via Identity · Rota base: /api/users

Visão geral
/api/users
IdentityUser
Objetivo: CRUD de usuários usando UserManager<IdentityUser> e ApplicationDbContext para update/delete customizados. Protege a exclusão do usuário principal definido em _settings.Theme.Email.
Endpoints
GET
Listar — retorna todos os usuários com envelope { data, recordsTotal, recordsFiltered }.
GET
Obter por id — /api/users/{id}, retorna IdentityUser.
POST
Criar — body form-data de IdentityUser; servidor gera Id e define UserName = Email. Define senha padrão Password123! (HACK indicado no código).
PUT
Atualizar — body form-data; usa _context.UpdateAsync(model, model.Id) (extensão de ApplicationDbContext).
DELETE
Remover — body form-data; bloqueia exclusão se model.UserName == _settings.Theme.Email. Usa _context.DeleteAsync<IdentityUser>(id).
Fluxo e validações
  • Listagem: AsNoTracking(); envelope DataTables.
  • Criação: senha default Password123! marcada como HACK; recomendável trocar por política segura e fluxo de convite/reset.
  • Delete guard: impede excluir usuário principal configurado em SmartSettings.
  • Update/Delete: usam extensões de ApplicationDbContext (UpdateAsync, DeleteAsync).
Segurança: Confirme autenticação/autorização aplicadas globalmente (não definidas neste controller). Avalie política de senha e restrições extras em produção.
Exemplo de requisição (cURL)
curl -X POST https://seu-host/api/users ^ -F "Email=novo.usuario@empresa.com" ^ -H "Authorization: Bearer <token>"

Troque o token e ajuste headers conforme o provedor de auth configurado.
